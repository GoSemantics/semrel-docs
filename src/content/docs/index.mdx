## Why semrel?

<CardGrid stagger>
  <Card title="Zero manual versioning" icon="approve-check">
    semrel analyses your [Conventional Commits](https://www.conventionalcommits.org/) and decides whether to bump `major`, `minor`, or `patch` — no human judgement required.
  </Card>
  <Card title="Fully pluggable" icon="puzzle">
    Every step of the pipeline is a swappable gRPC plugin. Swap out the GitHub provider for GitLab, replace the changelog renderer, or add a Slack notification — all without changing core semrel.
  </Card>
  <Card title="Monorepo ready" icon="seti:folder">
    Version multiple independent modules in a single repository, each with its own tag series and config file.
  </Card>
  <Card title="Language agnostic plugins" icon="translate">
    Plugins speak gRPC, so they can be written in Go, Python, Rust, Node.js — or any language with a Protobuf generator.
  </Card>
  <Card title="Dry-run mode" icon="information">
    Preview exactly what semrel would do before touching anything. Run `semrel release --dry-run` in CI to validate your config safely.
  </Card>
  <Card title="Open Source" icon="open-book">
    Apache 2.0 licensed. Built in the open at [GoSemantics/semrel](https://github.com/GoSemantics/semrel).
  </Card>
</CardGrid>
