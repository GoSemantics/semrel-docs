---
title: Official Plugins
description: Dedicated reference pages for all official SemRel plugins.
---

Official SemRel plugins are standalone Go binaries. Configure them through `args:` in `.semrel.yaml`, which SemRel exposes as `SEMREL_PLUGIN_*` environment variables.

You can also load plugin secrets from a `.env` file with `semrel --env-file .env release`. A successful plugin exits with status `0`; any non-zero exit code stops the pipeline.

## Official plugin catalog

| Plugin | Category | Description |
| --- | --- | --- |
| [`condition-generic`](/plugins/conditions/condition-generic/) | Condition | Runs a shell command and only passes when that command exits with status 0. |
| [`condition-gitea-actions`](/plugins/conditions/condition-gitea-actions/) | Condition | Confirms the current release is running inside Gitea Actions. |
| [`condition-github-actions`](/plugins/conditions/condition-github-actions/) | Condition | Confirms the current release is running inside GitHub Actions. |
| [`condition-gitlab-ci`](/plugins/conditions/condition-gitlab-ci/) | Condition | Confirms the current release is running inside GitLab CI. |
| [`hook-email`](/plugins/hooks/hook-email/) | Hook | Sends release notifications through SMTP after a SemRel run. |
| [`hook-gitplugin`](/plugins/hooks/hook-gitplugin/) | Hook | Pushes release-related changes to another Git repository or branch. |
| [`hook-jira`](/plugins/hooks/hook-jira/) | Hook | Updates Jira release metadata after SemRel publishes a version. |
| [`hook-matrix`](/plugins/hooks/hook-matrix/) | Hook | Posts release notifications to a Matrix room. |
| [`hook-slack`](/plugins/hooks/hook-slack/) | Hook | Posts release notifications to Slack through an incoming webhook. |
| [`analyzer-conventional`](/plugins/analyzers/analyzer-conventional/) | Analyzer | Determines the next SemVer bump from Conventional Commit messages. |
| [`analyzer-default`](/plugins/analyzers/analyzer-default/) | Analyzer | Determines the next SemVer bump by matching commit messages against regular expressions. |
| [`generator-changelog-html`](/plugins/generators/generator-changelog-html/) | Generator | Generates an HTML changelog for the pending release. |
| [`generator-changelog-md`](/plugins/generators/generator-changelog-md/) | Generator | Generates a Markdown changelog for the pending release. |
| [`generator-release-notes`](/plugins/generators/generator-release-notes/) | Generator | Builds concise release notes from the current release context and commit history. |
| [`updater-cargo`](/plugins/updaters/updater-cargo/) | Updater | Updates the version field in a Rust Cargo manifest. |
| [`updater-docker`](/plugins/updaters/updater-docker/) | Updater | Updates a version argument inside a Dockerfile. |
| [`updater-go`](/plugins/updaters/updater-go/) | Updater | Updates a Go source file that exposes the project version. |
| [`updater-gradle`](/plugins/updaters/updater-gradle/) | Updater | Updates the version key in a Gradle properties file. |
| [`updater-helm`](/plugins/updaters/updater-helm/) | Updater | Updates chart metadata in a Helm `Chart.yaml` file. |
| [`updater-homebrew`](/plugins/updaters/updater-homebrew/) | Updater | Updates a Homebrew formula with the new release URL and checksum. |
| [`updater-maven`](/plugins/updaters/updater-maven/) | Updater | Updates the version declared in a Maven `pom.xml` file. |
| [`updater-npm`](/plugins/updaters/updater-npm/) | Updater | Updates the version field in a `package.json` file. |
| [`updater-nuget`](/plugins/updaters/updater-nuget/) | Updater | Updates the version property inside a `.csproj` or other NuGet project file. |
| [`updater-python`](/plugins/updaters/updater-python/) | Updater | Updates Python package version metadata in `pyproject.toml` or a similar backend file. |
| [`updater-terraform`](/plugins/updaters/updater-terraform/) | Updater | Updates a Terraform variable that stores the application version. |
| [`provider-bitbucket`](/plugins/providers/provider-bitbucket/) | Provider | Publishes release information to Bitbucket from the SemRel release context. |
| [`provider-git`](/plugins/providers/provider-git/) | Provider | Creates git tags and optionally pushes branch updates through the local Git remote. |
| [`provider-gitea`](/plugins/providers/provider-gitea/) | Provider | Publishes releases to a Gitea instance. |
| [`provider-github`](/plugins/providers/provider-github/) | Provider | Publishes releases to GitHub using the generated SemRel tag, version, and changelog. |
| [`provider-gitlab`](/plugins/providers/provider-gitlab/) | Provider | Publishes releases to GitLab. |
